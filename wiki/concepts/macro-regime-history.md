"지금 미국 거시국면이 어디에 있고, 가장 닮은 과거 스토리가 어떻게
흘러갔는지, 오늘 그 스토리에 가까워졌는지 멀어졌는지"를 매일 추적하는
프레임. 사용자가 다른 대화에서 이미 구축한 방법론(G/I/L 산식, 1954~2026
원자료, 역사적 사례 매칭)을 이 위키의 SK하이닉스 데일리 리포트에 접목한
것 — [HBM Cycle Score](hbm-cycle-score.md)가 "SK하이닉스 미시 축"이라면
이 페이지는 그 위에 얹히는 "거시 배경 축"이다.

## 1. G/I/L 국면 점수

```
G(성장) = -(실업률_t − 실업률_t-12)        # 양수 = 개선
I(물가) = CPI YoY_t − CPI YoY_t-6           # 양수 = 물가 가속
L(유동성) = 연방기금금리_t − CPI YoY_t×100   # 양수 = 긴축적
```

4분면: G>0·I<0 = **recovery** / G>0·I≥0 = **overheat** / G≤0·I>0 =
**stagflation** / 그 외 = **reflation**

**역사적 유사 시기**는 1954~2026년 전체 월별 G/I/L을 z-score로
표준화한 뒤, 오늘과 유클리드 거리가 가장 가까운 과거 달들을 찾는
방식(최근 24개월은 자기상관 방지를 위해 후보에서 제외, 매칭된 시기끼리는
최소 12개월 간격 유지).

## 2. 데이터 파이프라인 (아키텍처)

```
sources/macro-database-1954-2026.md   ← 원자료(CPI/실업률/연방기금금리, 사용자 시드데이터)
        │
        ▼
scripts/regime_engine.py              ← G/I/L 계산 + 역사적 매칭 (파이썬, 라우틴이 bash로 직접 실행)
        │
        ▼
이 페이지("체크 이력")에 매일 결과 기록
```

**신규 월 데이터 반영 경로 — 웹검색과 API 두 가지 다 지원**:
- (A) 매일 아침 체크가 `python3 scripts/regime_engine.py`를 실행해 시드
  데이터의 최신월 기준 결과를 계산. 이 최신월이 실제 이번 달보다 오래됐으면
  (CPI/실업률/연방기금금리는 월 1회만 발표되므로), 웹검색으로 신규 발표치를
  확인해 `python3 scripts/regime_engine.py --append-month YYYY-MM --cpi X --unrate Y --fedfunds Z`로
  원자료에 새 행을 추가한 뒤 재계산.
- (B) 사용자가 **이미 다른 Claude Code 세션에서 FRED API 연동 파이프라인을
  운영 중** — 그쪽에서 더 정확한/실시간 수치가 나오면, 그 값을 그대로
  같은 `--append-month` 커맨드에 넣거나 sources/macro-database-1954-2026.md의
  CSV 블록에 직접 append하면 된다. 스크립트 입장에서는 값이 웹검색에서
  왔는지 API에서 왔는지 구분하지 않음 — **어느 경로로 넣어도 동일하게
  작동하도록 설계**. 값을 넣을 때 log.md에 출처(웹검색 vs API)를 남길 것.

시드 데이터 마지막 달인 **2026-05** 기준(6월 이후 발표치는 아직 미반영 —
다음 체크에서 웹검색으로 갱신 시도):

| G | I | L | 국면 | CPI YoY |
| --- | --- | --- | --- | --- |
| ≈0.00 (경계선) | +0.0151 | -0.62 | **stagflation** (경계선 판정) | 4.25% |

G가 거의 정확히 0이라 recovery/stagflation 경계에 걸쳐 있음 — "성장은
정체, 물가는 소폭 가속" 정도로 방향성을 과장하지 않고 유보적으로 해석.

**가장 닮은 과거 5개월** (거리순):

| 순위 | 시기 | 거리 | 국면 | 배경(일반 역사 지식 기반 — 다음 체크에서 웹검색으로 세부 검증) |
| --- | --- | --- | --- | --- |
| 1 | 1957-04 | 0.085 | overheat | 한국전쟁 이후 인플레 진정 국면이었으나, 이 몇 달 뒤(1957년 8월) 아이젠하워 리세션 시작 — "표면적 안정 뒤 침체 전조" |
| 2 | 1979-04 | 0.229 | overheat | 2차 오일쇼크 직전, 볼커 Fed 의장 취임(1979-08) 이전 — 이후 물가가 급격히 튀며 스태그플레이션 본격화 |
| 3 | 2002-12 | 0.339 | stagflation | 닷컴버블 붕괴 후 "고용 없는 회복"(jobless recovery), Fed는 이후 2003년 6월 1.0%까지 추가 인하 |
| 4 | 2005-09 | 0.343 | overheat | 주택시장 붐 절정기, 그린스펀 말기 긴축 사이클 막바지 — 2년 뒤 GFC 전조 |
| 5 | 2007-11 | 0.356 | stagflation | 유가 급등(배럴당 $90대)+성장 둔화 동시 진행, 한 달 뒤(2007-12) NBER 공식 침체 시작 |

**해석에 대한 솔직한 한계**: 사용자가 다른 대화에서 이미 도출했던 매칭
결과(1957-01·1956-12·1956-11·1968-01·2005-11)와 이번 계산 결과가 정확히
일치하지 않는다 — z-score 기준월/exclusion 규칙 차이일 가능성이 큼.
**같은 "1950년대 후반 + 2000년대 중반" 군집이 반복 등장한다는 방향성은
일치**하지만, 정확한 월 단위까지는 재현되지 않았음을 투명하게 밝혀둔다.
다음 체크에서 사용자가 원한다면 매칭 로직(exclusion window, z-score
기준기간)을 대화로 맞춰볼 수 있음.

## 4. 스토리 프로필 & 관전 포인트 (상위 3개 애널로그)

각 프로필은 확인지표(✅ 충족 시 스토리 강화)와 반박지표(❌ 충족 시 스토리
약화) 3~5개로 구성. **오늘은 최초 설정이라 아직 지표 판정을 안 했음 —
다음 체크부터 매일 확인.**

### ① 1957-04형 — "표면적 안정 뒤 침체 전조"
- 확인지표: 신용스프레드(하이일드) 확대 / 산업생산 둔화 / 소비심리 하락
- 반박지표: Fed 조기 완화 시그널 / 고용지표 견조 지속 / 신용스프레드 안정

### ② 1979-04형 — "긴축 초입, 물가 재가속 위험"
- 확인지표: 유가·에너지 가격 재급등 / Fed 긴축 장기화·추가 인상 / 기대인플레이션 상승
- 반박지표: Fed 조기 피벗(금리 인하 전환) / 유가 안정 / CPI 서프라이즈 하향

### ③ 2002-12형 — "고용 없는 회복, 저성장 장기화"
- 확인지표: 실업률 개선 정체 / 기업 설비투자 위축 / 저금리에도 신용확장 둔화
- 반박지표: AI·반도체발 CapEx 붐이 실업률 개선을 견인(지금 국면 특유의 변수) / 고용지표 서프라이즈 상회

### SK하이닉스 관점에서의 함의
1957-04형·2002-12형은 "메모리 수요 자체가 거시 침체로 훼손"되는 하방
시나리오, 1979-04형은 "물가·금리 재가속으로 밸류에이션 멀티플이
눌리는" 시나리오 — 셋 다 실현되면 [HBM Cycle Score](hbm-cycle-score.md)의
붕괴조건과 별개로 "매크로발 디레이팅" 리스크로 별도 취급해야 함.

## 5. 수렴도(convergence) 스코어 — 다음 체크부터 가동

```
score = Σ(확인지표 충족 +1) − Σ(반박지표 충족 -1)
convergence_pct = (score − min) / (max − min) × 100
```

정량 지표는 가능하면 FRED/웹검색으로 직접 확인, 정성 지표(유가 재급등
여부 등)는 최근 7일 뉴스 판단 + 근거 URL 기록. 오늘은 최초 설정이라
0%(중립) 기준선만 잡아두고, 다음 체크부터 실측 시작.
