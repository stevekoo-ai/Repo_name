"지금 미국 거시국면이 어디에 있고, 가장 닮은 과거 스토리가 어떻게
흘러갔는지, 오늘 그 스토리에 가까워졌는지 멀어졌는지"를 매일 추적하는
프레임. 사용자가 다른 대화에서 이미 구축한 방법론(G/I/L 산식, 1954~2026
원자료, 역사적 사례 매칭)을 이 위키의 SK하이닉스 데일리 리포트에 접목한
것 — [HBM Cycle Score](hbm-cycle-score.md)가 "SK하이닉스 미시 축"이라면
이 페이지는 그 위에 얹히는 "거시 배경 축"이다.

## 1. G/I/L 국면 점수

```
G(성장) = -(실업률_t − 실업률_t-12)        # 양수 = 개선
I(물가) = CPI YoY_t − CPI YoY_t-6           # 양수 = 물가 가속
L(유동성) = 연방기금금리_t − CPI YoY_t×100   # 양수 = 긴축적
```

4분면: G>0·I<0 = **recovery** / G>0·I≥0 = **overheat** / G≤0·I>0 =
**stagflation** / 그 외 = **reflation**

**역사적 유사 시기**는 1954~2026년 전체 월별 G/I/L을 z-score로
표준화한 뒤, 오늘과 유클리드 거리가 가장 가까운 과거 달들을 찾는
방식(최근 24개월은 자기상관 방지를 위해 후보에서 제외, 매칭된 시기끼리는
최소 12개월 간격 유지).

## 1-1. 거시지표 자동 수집 도구 (2026-07-24 밤 추가, 같은 날 밤 GitHub Actions로 전환)

## 2. 데이터 파이프라인 (아키텍처)

```
sources/macro-database-1954-2026.md   ← 원자료(CPI/실업률/연방기금금리, 사용자 시드데이터)
        │
        ▼
scripts/regime_engine.py              ← G/I/L 계산 + 역사적 매칭 (파이썬, 라우틴이 bash로 직접 실행)
        │
        ▼
이 페이지("체크 이력")에 매일 결과 기록
```

**신규 월 데이터 반영 경로 — 웹검색과 API 두 가지 다 지원**:
- (A) 매일 아침 체크가 `python3 scripts/regime_engine.py`를 실행해 시드
  데이터의 최신월 기준 결과를 계산. 이 최신월이 실제 이번 달보다 오래됐으면
  (CPI/실업률/연방기금금리는 월 1회만 발표되므로), 웹검색으로 신규 발표치를
  확인해 `python3 scripts/regime_engine.py --append-month YYYY-MM --cpi X --unrate Y --fedfunds Z`로
  원자료에 새 행을 추가한 뒤 재계산.
- (B) 사용자가 **이미 다른 Claude Code 세션에서 FRED API 연동 파이프라인을
  운영 중** — 그쪽에서 더 정확한/실시간 수치가 나오면, 그 값을 그대로
  같은 `--append-month` 커맨드에 넣거나 sources/macro-database-1954-2026.md의
  CSV 블록에 직접 append하면 된다. 스크립트 입장에서는 값이 웹검색에서
  왔는지 API에서 왔는지 구분하지 않음 — **어느 경로로 넣어도 동일하게
  작동하도록 설계**. 값을 넣을 때 log.md에 출처(웹검색 vs API)를 남길 것.

## 4. 스토리 프로필 & 관전 포인트 (상위 3개 애널로그)

**2026-07-24 갱신 — 공통점/차이점 + 양방향 시나리오 구조로 고도화**:
사용자 피드백 반영. 단순 확인/반박지표 나열에서, 각 애널로그마다
① 지금과 공통점 ② 지금과 결정적 차이점 ③ 앞으로 어느 쪽으로 전개되면
위험이 커지는지/작아지는지 ④ 어느 쪽으로 전개되면 상승 기대가
강해지는지/약해지는지 — 4개 축을 명시적으로 분리했다. **위험 축과
상승기대 축은 서로의 반대말이 아니다** — 예를 들어 거시침체 리스크가
낮아지는 것과 SK하이닉스 AI 사이클 상승 기대가 강해지는 것은 별개
질문이라, 따로 판단해야 헷갈리지 않는다. 오늘은 최초 설정이라 아직
지표 판정을 안 했음 — 다음 체크부터 매일 확인.

### ① 1957-04형 — "표면적 안정 뒤 침체 전조"

**공통점**: 물가는 진정된 듯 보이고(I는 소폭 가속에 그침), 고용 개선도
멈춰선(G≈0) 애매한 정체기라는 점 — 겉으로는 "나쁘지 않은" 지표들이지만
방향성이 안 보인다는 점이 닮았다.

**차이점**: 1957년엔 이렇다 할 구조적 성장 엔진이 따로 없었지만, 지금은
AI 데이터센터·HBM이라는 초대형 산업 사이클이 별도로 진행 중이다 — 거시
지표가 흔들려도 반도체 수요 자체는 산업 사이클의 자체 동력으로 버틸
여지가 1957년보다 훨씬 크다.

- 🔴 **위험이 커지는 방향**: 신용스프레드(하이일드)가 확대되고 산업생산·
  소비심리가 동시에 꺾이면 → 1957년처럼 몇 달 안에 실제 침체로 이어질 확률↑
- 🟢 **위험이 작아지는 방향**: 고용지표가 예상보다 좋게 나와 G가 다시
  플러스로 돌아서면 → "정체"에서 "완만한 회복"으로 재분류, 이 애널로그와의
  거리 자체가 멀어짐
- 🚀 **상승 기대가 강해지는 방향**: 거시가 흔들려도 하이퍼스케일러
  CapEx가 꺾이지 않고 유지되면(["경쟁생존논리"](sk-hynix-analyst-thesis-checkpoints.md#3-1-추적-프레임--경쟁-생존-논리-vs-roic-규율-2026-07-23-사용자-논의) 참고)
  → 거시 약세 속에서도 SK하이닉스만 예외적으로 버티는 디커플링 가능
- 📉 **상승 기대가 약해지는 방향**: 거시 침체 신호가 실제 데이터센터
  투자 둔화로 전이되면(2001년 닷컴버블처럼 반도체가 거시보다 먼저·더
  깊게 하락한 전례) → HBM 수요 자체가 훼손되는 최악의 조합

### ② 1979-04형 — "긴축 초입, 물가 재가속 위험"

**공통점**: 물가가 재가속하는 초기 국면이고, Fed 스탠스가 아직 명확히
매파로 굳어지진 않은 애매한 시점이라는 점.

**차이점**: 1979년은 오일쇼크라는 순수 공급충격이 물가를 밀어올렸지만,
지금은(만약 물가가 더 오른다면) AI 인프라 투자發 전력·설비 수요라는
수요측 요인일 가능성이 크다 — 원인이 다르면 대응도, 지속기간도 달라진다.
또 지금 Fed는 1970년대보다 훨씬 데이터 의존적이고 선제 커뮤니케이션을
하는 편이라, 정책 대응이 그때처럼 늦게 나올 가능성은 낮다.

- 🔴 **위험이 커지는 방향**: 에너지 가격이 실제로 급등하거나 Fed가
  긴축 사이클을 재개하면 → 1979-80형 스태그플레이션 재현 리스크,
  이 경우 SK하이닉스를 포함한 성장주 밸류에이션 멀티플 전반이 눌림
- 🟢 **위험이 작아지는 방향**: 물가 가속이 일회성 요인(예: 관세)으로
  확인되고 Fed가 동결 기조를 유지하면 → 긴축 우려 자체가 완화
- 🚀 **상승 기대가 강해지는 방향**: 물가 상승분이 AI 관련 설비투자·
  전력수요 확대로 확인되면(즉 "나쁜 인플레"가 아니라 "성장통") →
  오히려 메모리 수요 강세의 방증으로 재해석될 수 있음
- 📉 **상승 기대가 약해지는 방향**: 금리가 실제로 재인상되면 → 지금의
  강세 근거인 "저PER(6.8~6.9배)"는 그대로여도, 시장 전체 리레이팅 속도는
  둔화될 가능성

### ③ 2002-12형 — "고용 없는 회복, 저성장 장기화"

**공통점**: 저성장·고용 정체가 이어지는 국면이라는 점.

**차이점**: 2002년의 저성장은 닷컴버블 붕괴 후유증(설비 과잉, 부채
조정)이 원인이었던 반면, 지금은 그런 과잉투자 후유증이 없고 오히려
AI 투자가 고용·설비투자를 견인하는 정반대 방향으로 작용 중이다 — 이게
세 애널로그 중 지금 상황과 가장 근본적으로 다른 지점.

- 🔴 **위험이 커지는 방향**: AI CapEx 자체가 꺾이며 고용 견인력을
  상실하면 → 진짜 "고용 없는 회복"으로 수렴, 이 경우는 메모리 수요
  자체도 함께 훼손되는 복합 리스크
- 🟢 **위험이 작아지는 방향**: 반대로 AI투자가 계속 고용지표를
  예외적으로 지지하면 → 이 애널로그와의 유사성 자체가 옅어짐(멀어지는
  것 자체가 청신호)
- 🚀 **상승 기대가 강해지는 방향**: 위 "위험 작아지는 방향"과 사실상
  동일 — AI CapEx가 견조하게 유지되는 한 이 침체형 시나리오로 갈
  가능성 자체가 낮다는 게 곧 상승 기대 유지 근거
- 📉 **상승 기대가 약해지는 방향**: 별도 트리거 없음 — 위 "위험이
  커지는 방향"과 동일한 사건(AI CapEx 둔화)이 그대로 상승기대 훼손으로
  이어짐. 이 애널로그는 위험 축과 상승기대 축이 사실상 같은 방향으로
  움직이는 예외적인 케이스.

### SK하이닉스 관점에서의 함의

세 애널로그 모두에서 공통적으로 드러나는 건, **"AI CapEx 지속 여부"가
위험 축과 상승기대 축 양쪽을 동시에 좌우하는 스윙팩터**라는 점이다.
①·③번은 "메모리 수요 자체가 거시 침체로 훼손"되는 하방 시나리오,
②번은 "물가·금리 재가속으로 밸류에이션 멀티플이 눌리는" 시나리오로
성격이 다르지만, 셋 다 실현되면 [HBM Cycle Score](hbm-cycle-score.md)의
붕괴조건과는 별개로 "매크로발 디레이팅" 리스크로 취급해야 한다.

## 5. 수렴도(convergence) 스코어 — 다음 체크부터 가동

```
score = Σ(확인지표 충족 +1) − Σ(반박지표 충족 -1)
convergence_pct = (score − min) / (max − min) × 100
```

정량 지표는 가능하면 FRED/웹검색으로 직접 확인, 정성 지표(유가 재급등
여부 등)는 최근 7일 뉴스 판단 + 근거 URL 기록. 오늘은 최초 설정이라
0%(중립) 기준선만 잡아두고, 다음 체크부터 실측 시작.
