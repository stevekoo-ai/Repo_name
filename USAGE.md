# 사용설명서 (처음 쓰는 분용)

이 문서는 Claude Code와 GitHub을 처음 쓰는 사람을 기준으로 씁니다. 어려운 용어는
최소화하고, 실제로 화면에서 뭘 입력해야 하는지 그대로 적었습니다.

## 1. 이게 뭐 하는 물건인가요?

비유: **자료를 넣으면 알아서 정리 노트를 만들어주고, 나중에 물어보면 그 노트를
찾아서 답해주는 개인 비서**입니다.

- 원본 자료(글, 메모, 회의록 등)를 넣으면 → 비서가 요약/정리해서 "위키(wiki)"라는
  노트 묶음에 저장합니다.
- 나중에 궁금한 걸 물어보면 → 비서가 매번 원본을 처음부터 다시 읽는 대신, 이미
  정리해둔 노트를 찾아서 답합니다.
- 가끔 "정리한 노트들이 서로 모순되지 않는지" 점검도 시켜볼 수 있습니다.

이 세 가지 동작이 각각 `/ingest`(자료 넣기), `/query`(질문하기), `/lint`(점검하기)
명령어입니다.

## 2. 최소한만 알면 되는 용어

| 용어 | 뜻 |
|---|---|
| 저장소 (Repository, repo) | 이 프로젝트의 모든 파일이 들어있는 GitHub 상의 폴더. 여기서는 `Repo_name`. |
| 브랜치 (Branch) | 저장소 안의 "작업 버전" 하나. 지금 작업은 `claude/ai-agent-impl-002tip` 브랜치에서 하고 있어요. |
| 커밋 (Commit) / 푸시 (Push) | 변경한 내용을 저장하고 GitHub에 올리는 것. Claude가 알아서 해줍니다. 신경 쓸 필요 없음. |
| Pull Request (PR) | "이 브랜치의 변경사항을 진짜 메인 저장소에 합쳐도 될까요?"라고 올리는 제안서. 나중에 5번에서 설명. |
| 세션 (Session) | 지금 이 대화창 하나. |

## 3. 폴더 구조 한눈에 보기

```
sources/   ← 원본 자료를 넣는 곳 (글, 메모, 문서 등)
wiki/      ← 비서가 자동으로 정리해 만든 결과물 (건드릴 필요 없음, 자동 생성)
CLAUDE.md  ← 비서가 따르는 규칙 설명서 (평소엔 안 건드려도 됨)
```

## 4. 실제 사용법 (딱 3가지 명령만 알면 됩니다)

Claude Code 채팅창에 아래처럼 **그대로 입력**하면 됩니다.

### 4-1. 자료 넣기 — `/ingest`

**방법 A. 텍스트를 바로 붙여넣기** (파일 없이, 가장 쉬운 방법)

```
/ingest 여기에 정리하고 싶은 글이나 메모를 통째로 붙여넣으세요.
예: 오늘 회의에서 A프로젝트는 8월 말 출시로 결정, 담당자는 김철수...
```

**방법 B. 이미 있는 파일을 넣기**

```
sources 폴더에 "회의록-0713.txt"라는 파일로 이 내용을 저장하고 ingest 해줘: (내용 붙여넣기)
```

두 방법 다 결과는 같습니다. Claude가:
1. 내용을 `sources/` 폴더에 파일로 저장하고
2. `wiki/` 안에 정리된 노트를 새로 만들거나 기존 노트를 업데이트하고
3. 어떤 노트를 만들었는지/고쳤는지 알려줍니다.

### 4-2. 질문하기 — `/query`

```
/query A프로젝트 출시일이 언제야?
```

Claude가 `wiki/` 안의 노트를 찾아서, **어떤 노트를 근거로 답하는지**까지 알려줍니다.

### 4-3. 점검하기 — `/lint`

```
/lint
```

노트들끼리 모순되는 내용은 없는지, 빠진 링크는 없는지 점검해서 목록으로 보여줍니다.
(자동으로 고치지 않고, 먼저 보고만 합니다.)

## 5. 결과를 GitHub에서 직접 확인하는 법

1. 웹 브라우저에서 `https://github.com/stevekoo-ai/Repo_name` 접속 (로그인 필요)
2. 화면 왼쪽 위 "main"이라고 써진 드롭다운 버튼을 눌러서 `claude/ai-agent-impl-002tip`
   브랜치를 선택
3. `wiki` 폴더를 클릭하면 지금까지 정리된 노트 파일들을 볼 수 있습니다.

## 6. "이제 진짜로 확정" 하고 싶다면? (Pull Request)

지금 작업은 전부 `claude/ai-agent-impl-002tip`라는 별도 브랜치에 저장되고 있고,
아직 메인(`main`)에는 합쳐지지 않은 상태입니다. 합치고 싶으면 이렇게 말하면 됩니다:

```
Pull Request 만들어줘
```

그러면 Claude가 GitHub에 PR을 생성하고 링크를 드립니다. 그 링크를 열어서 내용을
확인한 뒤 "Merge" 버튼을 누르면 메인에 합쳐집니다. (이 버튼은 사람이 직접 눌러야
하며, Claude가 대신 누르지 않습니다.)

## 7. 자주 묻는 질문

**Q. 매번 CLAUDE.md를 읽으라고 말해야 하나요?**
아니요, `/ingest`, `/query`, `/lint` 명령 안에 이미 "규칙 파일을 따르라"는 지시가
들어있어서 자동으로 참고합니다.

**Q. API 키 같은 걸 발급받아야 하나요?**
아니요. 지금처럼 Claude Code 안에서 쓰는 동안은 필요 없습니다. (별도 프로그램으로
독립 실행하고 싶을 때만 필요한데, 지금은 해당 없음.)

**Q. 실수로 이상한 자료를 넣었어요.**
`sources/` 폴더의 파일은 원본이라 함부로 안 지우는 게 원칙이지만, "방금 넣은 자료랑
관련 노트 지워줘"라고 요청하면 Claude가 되돌려줍니다.

**Q. 꼭 영어로 자료를 넣어야 하나요?**
아니요, 한글로 넣어도 됩니다. 노트도 한글로 정리해달라고 요청할 수 있습니다.

## 8. 지금 바로 해볼 것 (실습 추천)

아래 문장을 그대로 복사해서 채팅창에 입력해 보세요:

```
/ingest 테스트 메모: 이 프로젝트는 Karpathy의 LLM Wiki 패턴을 구현한 것이고,
2026년 7월 13일에 처음 만들어졌다.
```

결과로 `wiki/` 안에 노트가 생기는 걸 확인한 뒤, 이어서:

```
/query 이 프로젝트는 언제 만들어졌어?
```

라고 물어보면 방금 넣은 내용을 근거로 답하는 걸 볼 수 있습니다.
